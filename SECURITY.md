# Política de seguridad

## Reportar una vulnerabilidad

Reporta vulnerabilidades de forma privada al responsable del repositorio por el
canal de contacto acordado con el equipo. Incluye una descripción, impacto,
pasos de reproducción y evidencia mínima. No publiques secretos, información
personal, URLs de documentos ni pruebas de concepto explotables en issues.

Se confirmará la recepción y se evaluará el reporte antes de publicar una
corrección. No existe un programa de recompensas.

## Alcance

La política cubre el backend Django, el cliente Expo, workflows de GitHub y la
configuración de despliegue almacenada en este repositorio. Los proveedores
externos (Railway, Mercado Pago, Expo, Apple y Google) deben reportarse también
por sus canales oficiales cuando corresponda.
